# CLAUDE.md

## 코딩 스타일

- 필요 이상으로 만들지 않는다 (YAGNI). 지금 안 쓰는 추상화, 나중을 위한 설정값, 한 곳에서만 쓰는 인터페이스는 넣지 않는다.
- 새 코드를 쓰기 전에 이미 있는 걸 재사용한다: 코드베이스 안의 helper/util/패턴 → 표준 라이브러리 → 이미 설치된 의존성 순으로 확인.
- 새 의존성은 마지막 수단. 몇 줄이면 되는 걸 위해 라이브러리를 추가하지 않는다.
- 짧은 diff가 이긴다. 되는 가장 단순한 해법을 채택하고, 그게 안 되는 이유가 명확할 때만 한 단계 올라간다.
- 주석은 WHY가 코드만으로 안 드러날 때만 (숨은 제약, 특정 버그 우회, 놀라운 동작). WHAT을 설명하는 주석은 쓰지 않는다.
- 실제로 일어날 수 없는 경우에 대한 에러 처리/검증/fallback은 추가하지 않는다. 신뢰 경계(사용자 입력, 외부 API)에서만 검증한다.
- 예외: 보안, 접근성, 신뢰 경계 입력 검증, 데이터 손실 방지용 에러 처리는 절대 생략하지 않는다.
